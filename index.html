How it works

The app fetches public repository signals (structure, docs, commits, languages) and turns them into recruiter-friendly insights.

1
Paste a repo URL
Enter any public GitHub repository link.
2
Fetch repository data
Metadata, file tree, README, language stats, and recent commit sample.
3
Generate report
Score + charts + strengths/risks + prioritized roadmap.

Metrics recruiters care about

Scores are computed across multiple dimensions to reflect real strengths and weaknesses of the repository.

Code quality

Heuristics from file composition, large-file risk, and structural signals.

Project structure

Signals like src/ separation, config files, clean layout, and conventions.

Documentation

README presence + key sections like setup, usage, features, and license.

Tests

Test folder/file detection and CI/test-config indicators.

Commit consistency

Commit recency and distribution (sample-based, fast and practical).

Real-world relevance

Signals like recency + basic popularity (stars/forks).
